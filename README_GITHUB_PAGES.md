# GitHub Pages Paper Trading

This project can run a serverless paper-trading dashboard on GitHub:

- GitHub Pages hosts `binance_lab.html`.
- GitHub Actions runs `scripts/github_paper_tick.mjs` every 5 minutes.
- The workflow writes `paper_status.json`, and the page reads that file in read-only mode.

This is not a real always-on trading server. It is a scheduled simulation loop. It does not place real Binance orders or use API keys.

## Files

- `.github/workflows/deploy.yml`: Pages deployment and 5-minute scheduled run.
- `scripts/github_paper_tick.mjs`: Binance Futures public-data scanner and paper-trading state updater.
- `public/paper_status.json`: generated by the workflow.
- `public/index.html`: generated from `binance_lab.html`.

## GitHub Setup

1. Push this folder to a GitHub repository.
2. In the repository, go to Settings -> Pages.
3. Set the source to deploy from the `gh-pages` branch.
4. Run the `Deploy Binance Paper Pages` workflow once manually.

After the first successful run, the page URL is shown in the workflow deploy step.

## Optional Workflow Settings

You can change behavior with repository variables or workflow env values:

- `PAPER_SYMBOLS`: comma-separated futures symbols.
- `PAPER_INTERVALS`: comma-separated intervals, default `3m,5m,15m`.
- `PAPER_CAPITAL`: simulated starting balance.
- `PAPER_LEVERAGE`: simulated leverage.
- `PAPER_MIN_SCORE`: entry threshold.
- `PAPER_MAX_POSITIONS`: max open positions.
- `BINANCE_API_BASES`: comma-separated alternate Binance Futures API base URLs.
